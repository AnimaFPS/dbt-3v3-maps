FOV and Focal Length Scaling Conversion

Here is a fov converter if you want the same fov in different games when your resolution aspect ratio is the same if you don't know the fov of your game use this tool to find it: https://www.desmos.com/calculator/3e9hs7usyr And for the Focal Length Scaling function the input and the output fov type must be the same and this assumes that there is no other factors in play for things like scopes and ads and is just between hipfire sens and fovs unless you have a game like apex, quake and kovaaks which have auto focal length functions for ads

Here is a video explaining why this works and why we need it


// Using jscalc.io with math.js library
// Key for variables
//  a = the fov output table
//  sixOHFOV = output 16:9 Horizontal FOV
//  fourOHFOV = output 4:3 Horizontal FOV
//  OFOV = output 1:1 Vertical FOV
//  OSENS = output sens from focal length scaling
//  IFOVT = input Fov type (values = 0.75, 0.5625 or 1 or the inverse ratio of horizontal/vertical resolution)
//  FOV = input FOV value
//  VRES = input Vertical resolution
//  HRES = input Horizonatal resolution
'use strict';
var IFOVT;
	
if( inputs.IFOVT > 0 ){
   IFOVT =  inputs.IFOVT;
} else {
  IFOVT = (inputs.VRES / inputs.HRES);
}
	
return {
  a: [{
	sixOHFOV: math.atan(((IFOVT)/(0.5625)) * math.tan(inputs.FOV * Math.PI/360)) * 360/Math.PI,
	fourOHFOV: math.atan(((IFOVT)/(0.75)) * math.tan(inputs.FOV * Math.PI/360)) * 360/Math.PI,
	OVFOV: math.atan(((IFOVT)/(1)) * math.tan(inputs.FOV * Math.PI/360)) * 360/Math.PI
},],
	OSENS: (math.tan(inputs.OFOV*Math.PI/360)/math.tan(inputs.FOV*Math.PI/360))*inputs.ISENS
};
